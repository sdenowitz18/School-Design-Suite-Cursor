# User story: Editing Public Academic Profile charts

## College Prep — metric definitions (read-only)

College Prep **metric names and structure are fixed** (e.g. users cannot rename “4-year graduation rate” or add/remove metric rows). Imported / canonical labels stay consistent with GreatSchools-aligned definitions. On **Current**, users may only edit **school values** (numbers) where the product allows, not the metric identity or chart line-up.

## Test Scores — subject names (read-only)

Test Scores **subject labels are fixed** (e.g. users cannot rename “Biology”). The subject list and order match the canonical / feed-aligned set. On **Current**, users may only edit **school %** values, not subject names. **Users cannot add subjects** on Race & Ethnicity or Low Income Test Scores tabs—the Overview + subject tabs are fixed to the feed-aligned set (same idea as the standalone Test Scores chart).

## Race & Ethnicity / Low Income — Test Scores “Overview” sub-tab (fully read-only)

Under **Test Scores**, the **Overview** sub-tab (rating-style view for all students and by group or low-income vs all) is **not editable** anywhere values appear. Users cannot change those overview metrics on **Current**. Editing, where supported, applies only under **subject-specific** sub-tabs.

## Race & Ethnicity / Low Income — “Student Progress” top tab (fully read-only)

The **Student Progress** tab (yearly growth ratings for all students and by race/ethnicity, or for all vs low-income) is **not editable** on **Current**. Same rule as Test Scores Overview: GreatSchools-sourced; users edit on other tabs where the product allows.

## Multi-tab edit mode

While **Edit** is active, switching **top-level** or **sub-** tabs within the same chart (e.g. Race & Ethnicity, Low Income Students) **keeps** edit mode so users can move between sections without losing the session. Changing the **school year** tab still exits edit mode. For **Students with Disabilities**, switching the **subject** tab while editing loads that subject into the form (unsaved changes on the previous subject are not kept unless the user saves).
